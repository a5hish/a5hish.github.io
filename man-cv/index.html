NOTES         top

      Following contains a list of my cases-studies and their finding : 
	
      1) Design and implementation of an image proccessing application.

      -- Findings :

         * Method overriding and abstraction can be achived in 'C' through 
	   function pointers (who says 'C' doesn't not supports OOPs ?). 
         * Shared Objects are used for creating dynamically loadable context 
	   for applications.    
         * Linker is a bundle of :
           1. Tool (ld on unix based system) - Used for static builds 
           2. Dynamic linker (in memory) - Used for dynamic builds  
         * Static builds are preferred if application have limited functionality 
	   which is fixed with a garantee of no changes for its life time.
         * Dynamic build are preferred when application is feature rich and is 
	   suppose to adapt future needs and must be supported for a long term.
 
      2) Design of Mutual-Exclusion locking API's
       
      -- Findings : 

        * Without atomic-intructions mutual-exclusion api's are difficult to 
	implement. 
        * semaphore are kernel resource with an atomic counter and a wait-queue 
	attached to it  
        * Mutual-Exclution is a protocol with type 
          1. Wait Lock (glibc implement this as mutex)
          2. polling Lock (glibc implement this as spin lock)
        * The following are realtime bugs assosiated with usage of locking.
          - Accidental release
          - Recrusive Deadlock
          - Owner-death deadlock
          - Recursive unlock
          - Re-initialization of live-lock
